# Git & GitHub Grading Rubric

Total: 40 points

## 1. Repo Baseline (8 pts)

- 7-8: First PR shows clean history, only course files, quick README or note for starting point.
- 5-6: Minor clutter or missing context but still reviewable.
- 3-4: Generated or irrelevant files present, baseline unclear.
- 0-2: Repo messy, slows review.

## 2. Commit Quality (12 pts)

- 11-12: Small logical commits with descriptive Conventional Commitâ€“style messages; history explains decisions.
- 8-10: Mostly good commits; occasional vague or oversized change.
- 5-7: Infrequent commits or messages with little info.
- 0-4: Sparse or confusing history.

## 3. Branch & PR Workflow (12 pts)

- 11-12: Feature branch for work, PR into your team repo with a clear summary, includes merge commits that document how the feature branch was integrated.
- 8-10: Branching and PR present but merge history or descriptions could improve.
- 5-7: Branch usage inconsistent or PR lacks useful context.
- 0-4: Work pushed to `main` or PR missing.

## 4. Team Collaboration (8 pts)

- 7-8: Both teammates commit, comment, and cross-review; PR summary lists who handled each part.
- 5-6: Shared effort visible but light on review notes or division of work.
- 3-4: One teammate dominates history or collaboration unclear.
- 0-2: No evidence of teamwork.
